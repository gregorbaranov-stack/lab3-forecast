# Анализ и прогнозирование статистических данных

Командный проект по лабораторной работе №3 «Системы контроля версий (VCS)»
дисциплины «Технологии программирования». Единое веб-приложение с общим
интерфейсом, объединяющее задачи двух участников команды; для каждой задачи —
таблица данных, интерактивные графики (масштабирование, выбор периода, экспорт)
и прогнозирование методом скользящей средней на `N` периодов вперёд.

## Выполненные задания

> Проект выполнен в режиме имитации команды (два аккаунта одного автора), что
> допускается заданием.

## Структура веток

- `master` — стабильная версия (итог слияния `dev`).
- `dev` — интеграционная ветка, принимает pull request'ы участников.
- `feature/currency`, `feature/temperature` — ветки участников (после слияния не удалялись).

## Архитектура

- **backend/** — Python + FastAPI. Бизнес-логика с применением ООП и SOLID:
  - `app/datasets.py` — абстрактный `Dataset` и реализации вариантов;
  - `app/statistics.py` — расчёт метрик варианта;
  - `app/forecasting.py` — стратегия скользящей средней;
  - `app/services.py` — `AnalysisService` и реестр вариантов `VARIANTS`;
  - `app/routers.py`, `app/main.py` — REST API.
- **frontend/** — React + TypeScript + Vite, графики на Plotly.
- **data/** — генератор синтетических данных и готовые наборы.

## Прогноз (скользящая средняя)

Прогноз следующего значения — среднее последних `n` значений ряда; полученный
прогноз добавляется в ряд и участвует в расчёте последующих (рекурсивная
скользящая средняя). Окно `n` и горизонт `N` задаются пользователем.

## Запуск

```bash
# Бэкенд (http://localhost:8000)
cd backend
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
.venv/bin/uvicorn app.main:app --reload --port 8000

# Фронтенд (http://localhost:5173)
cd frontend
npm install
npm run dev

# Перегенерация синтетических данных (по желанию)
cd data && python3 generate_data.py
```

## Удалённый доступ (через WireGuard)

Vite настроен слушать все интерфейсы (`server.host: true`) и проксировать `/api`
на локальный бэкенд. Поэтому наружу через туннель достаточно открыть **только
порт 5173** — бэкенд остаётся на `localhost:8000`.

1. Запустите бэкенд на localhost (как обычно): `uvicorn app.main:app --port 8000`.
2. Запустите фронтенд: `npm run dev`. Vite напечатает строку `Network:
   http://<IP>:5173/` — это адрес для доступа из сети/через WireGuard.
3. С удалённого устройства (подключённого к тому же WireGuard) откройте
   `http://<WireGuard-IP-этой-машины>:5173`. Запросы к API проксируются на
   бэкенд автоматически.

> Так как доступ идёт через приватный WireGuard-туннель, открывать порт бэкенда
> наружу и настраивать CORS не требуется.
